# JE Facemodeling — сайт

Адаптивный статический сайт без сборщика и без внешних библиотек.

## Как открыть

Самый простой вариант:

1. Откройте папку проекта в VS Code.
2. Установите расширение **Live Server**.
3. Нажмите правой кнопкой по `index.html` → **Open with Live Server**.

Можно также запустить из терминала:

```bash
python -m http.server 5500
```

Затем открыть `http://localhost:5500`.

## Где менять основные данные

- Основной текст и блоки: `index.html`
- Цвета, адаптивность, анимации: `styles.css`
- Телефон, WhatsApp, Instagram, Facebook и карта: `content.js`
- Логика меню, слайдеров, галереи и cookie: `app.js`

## Как добавить новую пару «до / после»

1. Подготовьте два фото одинакового размера.
2. Скопируйте их в `assets/results/`, например:
   - `client-2-before.webp`
   - `client-2-after.webp`
3. Откройте `content.js`.
4. Добавьте в массив `window.JE_RESULTS` объект:

```js
{
  title: 'Новый результат',
  before: 'assets/results/client-2-before.webp',
  after: 'assets/results/client-2-after.webp',
  altBefore: 'Клиент до процедуры',
  altAfter: 'Клиент после процедуры'
}
```

Карточка, интерактивный разделитель и полноэкранный просмотр появятся автоматически.

## Cookie и конфиденциальность

- Аналитика и рекламные пиксели сейчас не установлены.
- Google Maps не загружается до согласия пользователя.
- Выбор хранится локально в ключе `je_cookie_consent_v1`.
- Перед публичным запуском желательно проверить юридические тексты и полное имя/статус ответственного лица.

## Что заменить перед публичным запуском

- Заменить временную фотографию массажа на собственную горизонтальную фотографию процесса, когда она появится.
- Подтвердить окончательный список противопоказаний.
- Проверить точное написание имени и юридические реквизиты.
- Подтвердить, что все фотографии клиентов разрешено публиковать.
- При необходимости добавить латышскую версию.


## Временная публикация через GitHub Pages

1. Создайте публичный репозиторий на GitHub.
2. Загрузите **содержимое этой папки** в корень репозитория: `index.html`, `styles.css`, `app.js`, папку `assets` и остальные файлы.
3. Откройте `Settings` → `Pages`.
4. В `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main`, папку `/(root)` и нажмите `Save`.
6. Через несколько минут GitHub покажет публичную ссылку вида `https://USERNAME.github.io/REPOSITORY/`.

После следующих изменений достаточно снова загрузить/запушить файлы в ветку `main` — Pages обновит сайт автоматически.
