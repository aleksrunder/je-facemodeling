# JE Facemodeling — сайт RU / LV

Адаптивный статический сайт без сборщика и внешних библиотек. Русская версия открывается из `index.html`, латышская — из `lv.html`. Переключатель **RU / LV** находится в верхнем меню рядом с контактами.

## Как открыть локально

1. Откройте папку проекта в VS Code.
2. Установите расширение **Live Server**.
3. Нажмите правой кнопкой по `index.html` → **Open with Live Server**.

Или запустите в терминале:

```bash
python -m http.server 5500
```

Затем откройте `http://localhost:5500`.

## Где менять данные

- Русская версия: `index.html`
- Латышская версия: `lv.html`
- Цвета, адаптивность и анимации: `styles.css`
- Телефон, WhatsApp, Instagram, Facebook и карта: `content.js`
- Меню, слайдеры, галерея и cookie: `app.js`
- Политика конфиденциальности: `privacy.html` и `privacy-lv.html`
- Политика cookie: `cookies.html` и `cookies-lv.html`

## Как добавить новую пару «до / после»

1. Подготовьте два фото одинакового размера.
2. Скопируйте их в `assets/results/`, например:
   - `client-2-before.webp`
   - `client-2-after.webp`
3. Откройте `content.js`.
4. Добавьте объект в массивы `ru` и `lv` внутри `window.JE_RESULTS`.

Пример для русской версии:

```js
{
  title: 'Новый результат',
  before: 'assets/results/client-2-before.webp',
  after: 'assets/results/client-2-after.webp',
  altBefore: 'Клиент до процедуры',
  altAfter: 'Клиент после процедуры'
}
```

Карточка, интерактивный разделитель и полноэкранный просмотр появятся автоматически.

## Cookie и конфиденциальность

- Аналитика и рекламные пиксели сейчас не установлены.
- Google Maps не загружается до согласия пользователя.
- Выбор хранится локально в ключе `je_cookie_consent_v2`.
- Перед полноценным публичным запуском юридические тексты лучше проверить у специалиста по защите данных.

## Обновление сайта на GitHub Pages

Если репозиторий и GitHub Pages уже настроены:

1. Распакуйте новую версию сайта.
2. Замените в репозитории старые файлы новыми, включая новые файлы:
   - `lv.html`
   - `privacy-lv.html`
   - `cookies-lv.html`
3. Обязательно загрузите обновлённые `index.html`, `styles.css`, `app.js` и `content.js`.
4. Нажмите **Commit changes**.
5. Через 1–3 минуты обновите опубликованную ссылку с принудительной перезагрузкой: `Ctrl + F5`.

Если GitHub Pages ещё не настроен:

1. Создайте публичный репозиторий.
2. Загрузите **содержимое этой папки** в корень репозитория.
3. Откройте `Settings` → `Pages`.
4. В `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main`, папку `/(root)` и нажмите `Save`.

5. актуальную ссылку на Facebook — текущая ссылка ведёт на `bernuveikalscloverclothes`.

Тексты соответствуют текущей функциональности сайта: без форм, аналитики, Meta Pixel, онлайн-оплаты и встроенной системы записи. При добавлении любого из этих инструментов политику и баннер необходимо обновить до запуска.
